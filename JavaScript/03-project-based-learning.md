# Project-Based JavaScript Learning

## Why Project-First Learning Works
- You remember concepts longer
- You understand when to use each concept
- You gain confidence for real work

## Learning Projects (Simple to Advanced)

### Project 1: Smart Counter App
Concepts:
- Variables, functions, events, DOM updates

### Project 2: Todo with Filters
Concepts:
- Arrays, objects, localStorage, event delegation

### Project 3: Expense Tracker
Concepts:
- Form handling, CRUD in memory, persistence, summaries

### Project 4: Weather Dashboard
Concepts:
- Async/await, APIs, error handling, reusable functions

### Project 5: Notes App + Search
Concepts:
- Data structures, live filtering, modular code

### Project 6: REST API + Frontend
Concepts:
- Node.js, Express, routes, validation, fetch integration

### Project 7: Capstone (Choose One)
- E-commerce mini store
- Job application tracker
- Habit tracking platform
- Student learning dashboard

## Capstone Build Sequence
1. Requirements and feature list
2. UI wireframe and folder structure
3. Data model design
4. Build authentication (if needed)
5. Build CRUD modules
6. Add validation and errors
7. Add testing and linting
8. Deploy and document

## Definition of Done (Project)
- Works on desktop and mobile
- Handles invalid inputs safely
- Has loading/error states
- Has clean folder organization
- Includes README and setup steps

## Portfolio Advice
- Keep 3 polished projects
- Include one frontend, one backend, one full-stack
- Add screenshots + architecture diagram + live link
